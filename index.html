关键指标定义
计算逻辑与判断标准
# 核心库存公式 可用数 = 主仓实际库存 - 订单占有数 周转天数 = 可用数 / 日均销 库存周转 = 可用数 / 日均销 库+途周转 = (可用数 + 采购在途) / 日均销 # 补货公式 补货量 = (采购到货天数 + 15) × 日均销量 - 现有库存 - 采购在途 # 安全库存 安全库存 = 未来7天预估销量 × 1.5倍 # 爆款预警（影刀机器人） 预警条件 = 可用数 < 近七日销量 → 自动推送预警
指标健康阈值策略含义
周转天数≤15天 良好 / 16-30天 关注 / >30天 预警越低越好，反映库存消化速度
爆款库存低于7天销量 → 红色警报爆款永远不能断货
滞销红线当季SKU >90天 → 重大事故必须立即启动清货
补货红线周转天数 >30天 → 禁止补货先清后补

货盘策略
四类货品生命周期、白牌备帮策略、双星报货流程。白牌与双星分开定义，不能混用标准。
🏷
白牌策略标签
基于销售数据的动态分类
🔥 爆款▼

定义：稳定自然销售 + 白牌全渠道月销≥10000

动作：确保库存充足，优先补货。安全库存基数保持。采购在途可视化。

标准：全渠道近30天销量≥10000是核心指标。

📦 常备款▼

定义：近30天自然销量稳定>1000 + 稳定拿货推广

动作：维持合理库存，按周转补货。月销过千是备帮资格，不是固定备帮量。

🔄 流转款▼

定义：季节性/过渡款，带有流转月份标签

动作：控制库存，按时间节点清理。标签格式：26年XX月-流转款

💰 五元清▼

定义：低价清仓款

动作：立即停止补货，全力清仓。亏损不计入前端团队核算。

✨ 新品▼

定义：入库后未满60天的款式

动作：数据分析师每日监控。入库满60天必须变更策略（爆款/常备/流转/清货）。

🏷
双星策略标签
按总销量+近期表现的阶梯分类
🔥 爆款专项▼

定义：近7日销量≥2000

动作：优先保障，安全库存基数保持。有订单后申请辅料/织唛/LOGO。

📦 常备▼

定义：总销≥5000

动作：维持合理库存，按标准补货。

📊 平销▼

定义：总销2000-5000

动作：常规跟进，不超额备货。

✨ 新品▼

定义：有返单且<2000，60天内

动作：60天后评估升级为平销/常备或降级。

🆕 首单▼

定义：无返单，60天内

动作：60天后评估，不达标进入清仓。

🧹 清仓/五元清/私款▼

定义：滞销或终止合作款

动作：立即停止补货，全力清货。双星有订单后才可申请辅料/织唛/LOGO。

🔧
白牌备帮策略
提前备鞋帮半成品

策略：白牌爆款和常备款提前备鞋帮半成品，缩短生产周期。

触发条件：月销稳定过千的款具备备帮资格。

操作：与工厂协商提前生产鞋帮，按实际订单组装成品。

监控：帮子库存 > 现鞋 × 3 时，评估帮子利用率，考虑停做。

⚠️
季末所有款式优先控货、压库存、压帮量或收仓，不默认清货。季节性强的网鞋/拖凉鞋/棉鞋制定收仓计划。
📋
双星报货流程
前端报货群提需求

流程：前端运营在报货群提需求 → 采购部确认 → 有订单后申请辅料/织唛/LOGO。

关键节点：

  • 前端报货群提交款式、数量需求
  • 青岛总部审核、制作辅料/织唛/LOGO
  • 快递到位后才可生产双星帮面
  • 生产周期：飞织7-10天，复杂10-15天
💡
双星生产周期长于白牌，需预留更长时间窗口。

考勤制度
工作时间与请假规范

工作时间：上午 8:30-12:00，下午 13:30-18:00

旺季：冬季加班

淡季：单休

请假：钉钉发起事假申请 → 部门老大审核 → 超3天以上人事及老板审核

补卡：每月4次漏打卡机会，超出20元/次

迟到：第1次扣5元，第2次扣10元，第3次扣20元，第4次扣40元...累加

旷工：按日薪扣除，部门负责人连带

🎫
工牌管理制度
佩戴与保管规范

佩戴要求：进出办公室及仓库必须佩戴工牌。

遗失/损坏：及时到行政部办理挂失补办。

处罚：

  • 未佩戴工牌：5元/次
  • 故意损坏/涂改：10元 + 限期补办
  • 遗失未上报：50元
  • 转借他人：双方各50元
💰
财务报销
差旅费与费用报销

报销范围：购物样品、打车、付费工具、差旅费。

流程：取得合法票据 → 填写报销单 → 附上票据及付款截图 → 提交审批。

出差学习：报销前需提交《外出学习报告》。

支付方式：微信/支付宝/银行收款账号。

📚
新人培训
试用期与转正

试用期：一般前三个月为试用期。

培训内容：公司制度培训、岗位技能培训、通用能力培训。

转正：试用期满，发起转正流程，填写《转正审批表》。

提前转正：表现突出，经考核可缩短试用期。

生效时间：当月15日前发起，当月16日生效；15日后发起，次月1日生效。

清货制度奖惩
2025年4月4日实施

适用范围：开发部、采购部、前端销售（阿里业务主导）

关联部门：仓储部、采购部、开发部提供数据支撑

责任分工

  • 开发部：每月30号前提交2个月后新品到货计划
  • 采购部：收到计划后10工作日提供清货款式
  • 销售部：收到清货清单后5工作日出具清货计划

处罚标准

💰
延期或未提交：罚款100元/次

申诉机制

目标因市场波动、政策调整等客观原因未达成，需提交证据至总经办审核，通过后可调整考核结果。

品控驻厂制度
品质管控前置

品控固定驻核心工厂：对重点供应商实施驻厂质检。

驻厂职责：

  • 生产过程中的质量巡检
  • 原材料入库检验
  • 成品出厂前的抽检
  • 不合格品及时标记隔离

入仓质检：所有到货必须经过品控组（杨俊娜）入仓质检。

质检报告：每日出具质检日报，不合格品处理记录归档。

来货异常处理
质检不合格

发现异常：入仓质检发现不合格品。

立即动作：通知品控组（杨俊娜），暂停该批次入库。

原因分析：供应商生产问题？物流损坏？标签错误？

处理方案：

  • 轻微问题：返工处理后重新入库
  • 严重问题：整批退货，供应商承担损失
  • 重复问题：供应商评分降级，减少订单

归档：质检异常记录归档，纳入供应商评估。

升级路径
问题升级规则

一级（采购员）：日常补货、常规到货跟进、一般质检问题。

二级（采购主管-张伟晓）：爆款缺货、重大延迟、跨部门协调、清货审批。

三级（总经办-郭晓杰）：重大质量事故、供应商终止合作、制度申诉、大额付款审批。

紧急通道：涉及重大经济损失或客户投诉，可直接升级至老板。

入职第一周任务看板
7天内完成
Day 1
通读本行动指南
了解部门定位、组织架构、数据逻辑、流程框架。
Day 2
熟悉货盘表格
打开货盘看板，理解每个字段含义。练习筛选、排序、VLOOKUP。
Day 3
加入工作群
加入采购部群、业务沟通群、运营对接群、爆款预警群。
Day 4
学习补货流程
跟带教人学习一次完整补货流程：从数据监控到下单到跟进。
Day 5
独立巡看板
独立完成一次日常巡看板，记录发现的问题。
Day 6
学习清货流程
了解清货决策标准、月度循环、奖惩机制。
Day 7
1v1复盘
跟组长做1v1复盘，回顾本周学习，提出疑问，确认下周计划。
